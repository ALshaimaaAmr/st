Key Terms


1 command


-Serverless: Computing model. No server management. Run code on
demand.


-FaaS: Function as a Service. Core of serverless. Run single functions in
cloud.


-Event-driven: Architecture triggering functions based on events.


-Pay-per-use: Billing model. Pay only for resources used.


-Cold start: Initial delay when function starts from scratch.


-Warm start: Faster execution using pre-warmed instance


-Concurrency: Multiple function instances running simultaneously.


-Stateless: Functions maintain no persistent state between invocations.
